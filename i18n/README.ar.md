> مستودع الورقة البحثية Event-based Rapid Organoid Imaging.

## نظرة عامة

هذا المستودع يُستخدم حاليًا كمستودع مرافق للورقة البحثية **Event-based Rapid Organoid Imaging**.

في وقت إعداد هذه المسودة، يحتوي المستودع على الترخيص وقواعد التجاهل وبنية توثيق أولية، لكنه لا يتضمن بعد أي شيفرة مصدر متتبعة أو حزم أو سكربتات أو أصول تجريبية.

| لمحة | الحالة الحالية |
|---|---|
| النطاق | مستودع مرافق للورقة |
| الأثر الأساسي | هيكل توثيقي أولي |
| توفر الشيفرة | لا توجد ملفات تنفيذ متتبعة حتى الآن |
| الجاهزية متعددة اللغات | يوجد المجلد `i18n/` |

## الميزات

- هيكل مشروع أساسي مرجعي للورقة: *Event-based Rapid Organoid Imaging*.
- يتضمن ترخيص Apache 2.0.
- قالب `.gitignore` موجّه لبايثون، ما يشير إلى أدوات بايثون محتملة مستقبلًا.
- وجود مجلد `i18n/` لنسخ README متعددة اللغات.

## بنية المشروع

```text
RapidOrganoidImaging/
├── README.md
├── LICENSE
├── .gitignore
├── i18n/
└── .auto-readme-work/
```

ملاحظات:
- `.auto-readme-work/` مساحة عمل لخط أنابيب المعالجة، وقد تحتوي على ملفات مساعدة مُولّدة.
- الملفات المتتبعة في جذر المستودع حاليًا هي `README.md` و`LICENSE` و`.gitignore`.
- يوجد `i18n/`، لكن قد تكون ملفات README الخاصة بكل لغة لا تزال قيد الإنشاء.

## المتطلبات المسبقة

الحد الأدنى الحالي للمتطلبات:

- `git` (لاستنساخ المستودع ومتابعة التحديثات)
- اختياري: أدوات بايثون (`python`, `pip`, virtual environments) عند/إذا تمت إضافة شيفرة

لا توجد تبعيات تشغيل إلزامية مُعلنة بعد ضمن ملفات manifest المتتبعة.

## التثبيت

```bash
git clone <your-repository-url>
cd RapidOrganoidImaging
```

إذا/عند إضافة شيفرة بايثون، قد يكون إعداد البيئة المعتاد كما يلي:

```bash
python -m venv .venv
source .venv/bin/activate
```

ملاحظة افتراضية: لم يتم تحديد أوامر تثبيت التبعيات بعد، لأنه لا يوجد حاليًا ملف manifest للتبعيات ضمن الملفات المتتبعة.

## الاستخدام

الاستخدام الحالي يركّز على التوثيق أولًا:

- قراءة نطاق المشروع والتحديثات في `README.md`.
- الرجوع إلى عنوان الورقة المرتبط: **Event-based Rapid Organoid Imaging**.
- متابعة الالتزامات القادمة التي تضيف سكربتات قابلة لإعادة الإنتاج أو مجموعات بيانات أو شيفرة نماذج/استدلال.

نظرًا لعدم وجود نقاط دخول تنفيذية متتبعة حتى الآن، لا يوجد أمر تشغيل يمكن توفيره في هذه المرحلة.

## الإعدادات

لا توجد ملفات إعدادات محددة حاليًا (على سبيل المثال، لا يوجد `.env.example` أو `config.yaml` أو مخطط معاملات CLI ضمن الشيفرة المتتبعة).

عند تقديم الإعدادات لاحقًا، ينبغي أن يوثق هذا القسم:

- متغيرات البيئة المطلوبة
- مسارات البيانات/المدخلات
- مواقع المخرجات
- خيارات العتاد/زمن التشغيل

## أمثلة

لا توجد أمثلة تنفيذية مُضافة بعد.

فئات أمثلة مخطط لها (سيتم تعبئتها عند توفر الشيفرة):

- تحضير البيانات / التحميل
- معالجة تدفق الأحداث
- سير عمل الاستدلال أو التحليل لتصوير الأورجانويدات
- عرض النتائج بصريًا/تصديرها

## ملاحظات التطوير

- ملف `.gitignore` الحالي قالب شامل لبايثون، ويتجاهل بالفعل ملفات virtualenv/build/test الشائعة.
- لا يوجد بعد سير عمل CI أو إعدادات formatter أو مجموعة اختبارات متتبعة.
- المجلد `i18n/` موجود وجاهز لملفات README الخاصة بكل لغة.

## استكشاف الأخطاء وإصلاحها

### قمت باستنساخ المستودع ولكن لا توجد شيفرة

هذا متوقع في الحالة الحالية للمستودع. يعمل المستودع حاليًا كهيكل مرافق للورقة البحثية.

### روابط اللغات تشير إلى ملفات قد لا تكون موجودة بعد

قد يحدث هذا قبل إنشاء ملفات README متعددة اللغات. يتم الإبقاء على سطر التنقل بين اللغات عمدًا للحفاظ على بنية i18n متسقة.

### لا أستطيع العثور على متطلبات التثبيت

لا يوجد حتى الآن ملف manifest للتبعيات ضمن الملفات المتتبعة (مثل `requirements.txt` أو `pyproject.toml` أو `environment.yml`).
